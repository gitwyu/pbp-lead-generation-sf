# Project by Project (PbP) Lead Generation Pipeline

## Overview
This project is a modular toolset designed to automate the discovery and enrichment of corporate sponsors and product donors for [Project by Project (PbP)](https://www.projectbyproject.org/). This script was created by the **PbP NYC chapter** to streamline our fundraising outreach for our annual Plate by Plate event, and we are sharing it here to help out the LA and SF chapters with their own local campaigns!

Currently, this pipeline relies exclusively on the Gemini API, its built-in tools (like Google Search Grounding), and a multi-agent architecture to find target companies and locate the best point of contact (CSR, PR, or general info).

## Notes on (Un)reliability of AI Agent Output
While we've done our best to optimize our prompts and make our agents double check their research, there will still be occasional hallucinations (i.e. wrong person with a name, outdated info such, made up emails, etc.). Please exercise your own verification before pursuing any outreach!

Each row contains several useful fields to kick off your manual verification:
* `Source`: a list of links the agents used during research
* `Contact 1/2 LinkedIn`: check their LinkedIn pages to see if they still work there as of today, and if they actually have that job title

## Prerequisites & Environment Setup

This project uses Python. Follow these steps to set up your environment before running the agents:

### 1. Create a Virtual Environment
It's highly recommended to use a virtual environment to keep dependencies isolated cleanly.
```bash
# Create a new virtual environment named 'venv'
python3 -m venv venv

# Activate it (Mac / Linux)
source venv/bin/activate

# Activate it (Windows)
venv\Scripts\activate
```

### 2. Install Dependencies
Once activated, install the required packages using the provided `requirements.txt`:
```bash
pip install -r requirements.txt
```

### 3. Get a Gemini API Key
You will need a Gemini API key. We use Gemini's built-in Google Search grounding tool to perform web searches and information extraction.
- To get a key, head over to Google AI Studio: [https://aistudio.google.com/app/apikey](https://aistudio.google.com/app/apikey)
- Create a new API key.

### 4. Set Up Your Environment Variables
Store this newly created key in a `.env` file in the root directory of this project:
```env
GEMINI_API_KEY=your_gemini_api_key_here
```

## Pipeline Architecture
The system is broken down into two sequential Python agents:

1. **`discovery_agent.py`**: Queries search APIs and performs keyword web searches to build target lists across categories: Fortune 500 companies, mid-to-large businesses, and AAPI-owned consumer brands. It outputs a raw list of companies, tagged with industry and donation viability (e.g., Monetary vs. Product In-Kind).
2. **`enrichment_agent.py`**: Takes the output from the discovery agent and uses **Gemini with Google Search Grounding** to find email addresses. It queries the web for specific PR, Charity, or CSR contacts for the company. It falls back to locate the general `info@`/`hello@` email if specific departments aren't found. It *must* record the `Source URL` where the contact was found.

## Running the Pipeline

You will run this process in two sequential steps. By default, the scripts might run in mock mode if you don't pass the appropriate flags, so use `--nomock` to hit the real Gemini API.

### Step 1: Run the Discovery Agent
This script searches the web to build a list of target companies based on the queries pre-configured in the script.

```bash
COMPANIES_FILE=sample_data/companies.csv # specify your own here
python discovery_agent.py --output $COMPANIES_FILE --limit 2 --verbose --num_parallel 5 --nomock
```
*   `--output`: Where to save the initial list of discovered companies.
*   `--limit`: Maximum number of websites to pull per search query.
*   `--num_parallel`: How many async workers to use (speeds up the process significantly).
*   `--nomock`: Disables local mock data and actually calls Gemini.


### Step 2: Run the Enrichment Agent
This script takes the discovered companies from Step 1, reads their websites, and finds the best points of contact (CSR, Philanthropy, or ERG leads).

```bash
COMPANIES_FILE=sample_data/companies.csv # specify your own here
ENRICH_FILE=sample_data/companies_enriched.csv # specify your own here
python enrichment_agent.py --input $COMPANIES_FILE --output $ENRICH_FILE --verbose --num_parallel 5 --nomock
```
*   `--input`: The CSV generated by Step 1.
*   `--output`: Where to save the final enriched list.
*   `--num_parallel` & `--nomock`: Same behaviors as above.

*(Note: The enrichment agent also supports `--resume` if you need to stop your script halfway through and want to pick up where you left off without spending extra tokens!)*

### Cost Estimation
We provide a rough API cost estimate below of running with `--model gemini-3-flash-preview` (the default), to obtain the NYC team's list of 736 companies and fundraising leads.
For, `discovery_agent.py` we used `--limit 50` with the 16 queries of `DISCOVERY_PLAN`. `enrichment_agent.py` then researches for all rows. We estimate the cost to be $125-150 for a similar sized run, but results may vary.

To do your own estimate, you can do your own run with `--limit 2` (use new CSV paths above), wait for the GCP billing to show the cost, then multiply by 25.


### Output Schema
| Column | Description |
|---|---|
| `Company Name` | Full legal or trade name |
| `Website` | Primary website URL |
| `HQ Location` | City, State (prioritize NY-area and national brands) |
| `Industry / Category` | e.g., Cosmetics, F&B, Tech, Fashion, Media |
| `Company Type` | See tags below |
| `AAPI Relevance` | `High` / `Medium` / `Low` |
| `AAPI Notes` | Brief justification (AAPI-founded, AAPI-focused products, etc.) |
| `Contact 1/2 Name` | Full name if found, else blank (focus on mid-level/ERG contacts for large cos) |
| `Contact 1/2 Title` | e.g., "Manager of Corporate Social Responsibility" |
| `Contact 1/2 Email` | Best available email |
| `Contact 1/2 LinkedIn` | Contact LinkedIn if found |
| `General Email` | e.g., partnerships@, foundation@, info@ |
| `General Contact Webpage` | The URL of the contact form or CSR page |
| `Source` | Where the info was found (URL or tool) |
| `Date Sources Updated` | The date each source was updated |
| `Notes` | Anything else useful for outreach. Always include contact confidence level here (e.g., "Contact confidence: high") |
| `Date Added` | Automatically assigned date when discovery agent found the lead |
